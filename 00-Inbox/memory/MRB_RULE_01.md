**MRB Capture Rule**

Every completed chat session that contains durable knowledge, decisions, insights, system changes, workflows, or project progress must be summarized as a **Memory Restoration Block (MRB)**.

Execution requirements:

1. Package the session using **MRB_v1_TEMPLATE.md**.
2. Assign a unique MRB title and metadata.
3. Create appropriate links to related notes, projects, systems, and previous MRBs.
4. Register the MRB in **MRB_INDEX.md**.
5. Store the finalized MRB in:

`D:\Users\RSky\Desktop\AI.ProFit\rsky.crv\04-Claude\memory\mrb`

6. Ensure the MRB is discoverable through both:

   * MRB_INDEX.md
   * Obsidian backlinks and internal links

The MRB becomes the authoritative compressed memory artifact for the session and must preserve enough context to restore the reasoning, decisions, and outputs of the original conversation.
